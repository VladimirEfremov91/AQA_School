Летим на отдых! Но что бы багаж с любимыми плавками и очками авиаторами доехал - надо что бы ПО в аэропорту работало как часы.
В аэропорту есть стойка приема багажа.

Пассажир подходит к стойке и хочет сдать чемодан. Программа должна проверить:

- существует ли рейс
- не слишком ли тяжелый багаж
- корректно ли переданы данные пассажира
- корректно ли указан вес багажа
- не сломался ли принтер багажных бирок
- не произошла ли катастрофа с багажной лентой

Часть 1. Создай класс BaggageTicket

Это багажная бирка, которую получает пассажир после успешной сдачи багажа.
Объяви приватные поля:
- имя пассажира
- номер рейса (может содержать буквы и цифры)
- вес багажа (целое число)


Добавь:
- конструктор
- геттеры
- переопределенный toString()

Пример создания объекта:
new BaggageTicket("Ivan Petrov", "SU-123", 18);


Часть 2. Создай иерархию проверяемых исключений

Создай базовое проверяемое исключение: public class AirportServiceException
От него должны наследоваться:

FlightNotFoundException - указанного рейса нет в списке доступных рейсов
OverweightBaggageException - багаж слишком тяжелый
BaggageTagPrintException - не получилось напечатать багажную бирку

Эти исключения должны быть проверяемыми, потому что вызывающий код может нормально на них отреагировать, например:
попросить пассажира проверить номер рейса
отправить пассажира на доплату за перевес
проверить состояние принтера (есть чернила, есть лента бирок) напечатать бирку еще раз


Часть 3. Создай непроверяемые исключения

Создай исключения:
InvalidPassengerNameException (имя пассажира null или пустое)
InvalidBaggageWeightException (вес багажа меньше или равен нулю)
Эти ситуации считаем ошибками программиста, поэтому делаем исключения непроверяемыми.


Часть 4. Создай свой Error

Создай класс: ConveyorBeltMalfunctionError, унаследуй его от Error
Это не обычная ошибка бизнес-логики, а критическая ситуация, блокирующая работу пункта приема багажа.
В обычной программе такие ошибки не нужно ловить через catch.


Часть 5. Создай класс BaggageDropDesk

Внутри класса должно быть поле - массив доступных рейсов. Добавь конструктор, инициализирующий это поле.
Добавь метод для сдачи багажа. Метод должен принимать имя пассажира, номер рейса и вес багажа. Метод должен:

- Проверить имя пассажира. Если имя пассажира null или имя пустое ("") - выбросить InvalidPassengerNameException
- Проверить вес багажа. Если вес багажа отрицательный - выбросить InvalidBaggageWeightException
- Проверить наличие рейса. Если рейса нет в массиве базы данных аэропорта - выбросить FlightNotFoundException
- Проверить вес багажа. Если вес больше 23 кг - выбросить OverweightBaggageException
- Имитировать проблему с печатью багажной бирки. Например, если номер рейса равен "AE-404" - выбросить BaggageTagPrintException

Если все хорошо - вернуть из метода новый BaggageTicket и оповестить пассажира о необходимости уплатить налог за слишком правильные введенные данные.


Часть 6. Проверь работу в main

Создай массив рейсов, например такой:

String[] flights = { "SU-123", "TK-777", "KC-909", "AE-404" };

Создай стойку приема багажа BaggageDropDesk. Проверь в main работу по разным сценариям:
1. успешная сдача багажа
2. указанного пассажиром рейса не существует в базе
3. багаж слишком тяжелый
4. проблема с печатью бирки
5. некорректное имя пассажира (null)
6. некорректный вес багажа (отрицательный)

Обработка возникающих исключений должна быть произведена в try-catch с множественным catch блоком - т.е. на каждый тип возникшего в try исключения - свой обработчик.

Часть 7. Отладка

Проследите за ходом возникновения одного из исключений от момента его выброса (нужно поставить breakpoint на нужной строке с throw) и до попадения в соответствующий обработчик - блок catch (еще один breakpont).

В качестве пруфов своего продвинутого владения отладчиком - пришлите партнеру по кросс ревью скрины экрана с открытым отладчиком в двух ситуациях выше - 1. момент создания исключения, 2. момент его перехвата
