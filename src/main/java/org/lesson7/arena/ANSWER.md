1. Почему heroesCreated должно быть статическим?
- Это нужно для того, чтобы оно было в единственной реализации, за счет чего достигается правильность подсчета количества 
созданных героев.

2. Почему name, health и level не должны быть статическими?
- Это нужно для того, чтобы в каждом экземпляре есть свои реализации этих полей.

3. Почему printHeroesCreated() можно вызвать без создания объекта?
- Потому что это статитеский метод класса, который вызывается без создания объектов.

4. Можно ли из статического метода напрямую обратиться к нестатическому полю name? Почему?
- Нельзя, потому что статический метод относится ко всему классу целиком, а не к конкретному объекту.

5. Почему проект перестанет компилироваться, если пометить класс Hero модификатором final?
- Потому что из него нельзя будет создавать классы-наследники, которые есть по условиям задания на этот проект.

6. Почему переменные имеют тип Hero, но при этом вызываются версии метода attack() из конкретных наследников?
- потому что объекты в этих переменных являются конкретными наследниками.

7. Почему переменная knight имеет модификатор final, но ее поля могут быть изменены? 
- Потому что в данном случае мы меняем состояние объекта.
