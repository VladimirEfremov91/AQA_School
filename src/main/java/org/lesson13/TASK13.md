Вы — главный IT-специалист сопротивления. Ваша задача — разработать систему для координации штурма, учёта пришельцев и управления отрядами.

Часть 1: База данных пришельцев

Создайте класс Alien (файл Alien.java) с полями:

String name — имя пришельца (например, «Зигмунд»)
String planet — планета происхождения
int dangerLevel — уровень опасности (1–10)
Требования к классу Alien:
Переопределите equals() и hashCode(), чтобы пришельцы считались одинаковыми при совпадении имени и планеты.
Переопределите toString() для удобного вывода информации об объекте.
Логика в main:
Создайте ArrayList<Alien>.
Добавьте 5 объектов Alien. Два из них должны иметь одинаковые имя и планету, но разный dangerLevel.
Проверьте: содержит ли список дубликат.
Выведите результат на экран.

Часть 2: Формирование отрядов

Создайте класс SquadManager (файл SquadManager.java), который будет отвечать за работу с отрядами. Требования к классу SquadManager:
Метод demonstrateListCreations(), внутри которого:

Создайте список new ArrayList<>() — основной отряд. Добавьте 4 имени.
Создайте список Arrays.asList() — отряд поддержки. Создайте сразу с 3 именами.
Создайте список List.of() — элитный отряд. Создайте сразу с 2 именами.
В этом же методе попробуйте добавить и удалить по одному штурмовику из каждого списка.
Оберните каждую операцию добавления/удаления в try-catch и выведите результат: успех или название перехваченного исключения.

Логика в main:
Создайте экземпляр SquadManager и вызовите метод demonstrateListCreations().

Часть 3: Отсеивание трусов

Добавьте в класс SquadManager метод filterOutCowards(List<String> squad).
Требования к методу filterOutCowards:
Выведите отряд до фильтрации.
Используя Iterator, удалите из переданного списка всех штурмовиков, чьи имена начинаются на «Трус».
Выведите отряд после фильтрации.
Бонус: реализуйте альтернативную фильтрацию через removeIf() (можно закомментировать или вынести в отдельный метод).

Логика в main:
Создайте ArrayList<String> с 3 обычными именами и 2 именами на «Трус» (например, «Трус Вася»).
Передайте этот список в метод filterOutCowards.

Часть 4: Очередь на вход

Создайте класс AssaultQueue (файл AssaultQueue.java).
Требования к классу AssaultQueue:
Внутри класса объявите поле LinkedList<String> queue.
Реализуйте методы:
addRecruit(String name) — добавляет штурмовика в конец очереди.
retreatCoward() — удаляет и возвращает штурмовика из начала очереди.
printQueue() — выводит текущее состояние очереди.
Переопределите toString() для вывода информации об очереди.
Логика в main:
Создайте объект AssaultQueue.
Добавьте 5 человек.
Имитируйте уход 2 человек из начала очереди (выведите их имена).
Добавьте 3 новых в конец.
Выведите итоговое состояние очереди.

Часть 5: Отчёт командованию

Создайте класс MissionReport (файл MissionReport.java) с полями:

String missionName
List<Alien> capturedAliens
int squadSize
Требования к классу MissionReport:
Переопределите toString(), чтобы вывод был читаемым, например:
Миссия: Штурм Зоны 51
Поймано пришельцев: 3
Размер отряда: 50
Логика в main:
Создайте объект отчёта и выведите его на экран.
Создайте второй отчёт с теми же данными.
Сравните их через == и через equals(). Выведите результаты обоих сравнений.
